摘要

GitHub Pages 是由 GitHub 官方提供的一种免费的静态站点托管服务，让我们可以在 GitHub 仓库里托管和发布自己的静态网站页面。

GitHub Pages 是一种静态站点托管服务，它直接从 GitHub 上的存储库获取 HTML、CSS 和 JavaScript 文件，可以选择通过构建过程运行文件，并发布网站。

GitHub 页面网站的类型

GitHub 页面网站有三种类型：项目、用户和组织。项目网站连接到 GitHub 上托管的特定项目，例如 JavaScript 库或配方集合。用户和组织站点连接到 GitHub.com 上的特定帐户。

要发布用户站点，您必须创建由您的个人帐户拥有的名为 的存储库。要发布组织站点，必须创建名为 的组织拥有的存储库。除非您使用的是自定义域，否则用户和组织站点可在 或 上找到。<username>.github.io``<organization>.github.io``http(s)://<username>.github.io``http(s)://<organization>.github.io

项目网站的源文件与其项目存储在同一存储库中。除非使用的是自定义域，否则项目网站可在 或 中找到。http(s)://<username>.github.io/<repository>``http(s)://<organization>.github.io/<repository>

您只能在 GitHub 上为每个帐户创建一个用户或组织站点。项目网站（无论是由组织还是个人帐户拥有）都是无限制的。

静态站点生成器

GitHub 页面发布您推送到仓库的任何静态文件。您可以创建自己的静态文件或使用静态站点生成器为您构建站点。还可以在本地或其他服务器上自定义自己的生成过程。
如果您使用自定义构建过程或 Jekyll 以外的静态站点生成器，则可以编写 GitHub 操作来构建和发布您的网站。

如果您从源分支发布站点，GitHub Pages 默认情况下将使用 Jekyll 来构建您的网站。如果要使用 Jekyll 以外的静态站点生成器，我们建议您编写 GitHub 操作来构建和发布站点。否则，通过创建一个在发布源根目录中调用的空文件来禁用 Jekyll 构建过程，然后按照静态站点生成器的说明在本地构建站点。. Nojekyll

GitHub Pages 不支持服务器端语言，如 PHP、Ruby 或 Python。
